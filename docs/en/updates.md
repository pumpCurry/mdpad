## v1.1.00039 (2026-02-23)

### Git Integration
- **Git Diff Mode**: Rich Diff (Preview pane) dropdown gains "vs. Git HEAD" option. When the current file is tracked by git, you can compare your edits against the last committed version (`git show HEAD:<file>`)
- **Git Status Bar**: When the file is inside a git repository, the status bar shows: repository name, branch name, short commit hash, and commit count on the branch
- Git availability, repository detection, and file tracking are checked automatically; all git-related UI is hidden gracefully when not applicable
- Uses `execFileSync` (not `execSync`) for shell-injection safety; all git commands run with `windowsHide: true` and 5-second timeout
- Git info is cached per file with a 5-second TTL; cache is invalidated on save

### Rich Diff Fix
- **Block-level diff**: Replaced line-level `diffLines()` with block-level `diffArrays()`. Markdown is split into semantic blocks (tables, fenced code blocks, Mermaid diagrams, blockquotes, lists, paragraphs) before diffing, so multi-line structures render correctly in the Rich Diff preview
- Tables, Mermaid diagrams, and code blocks no longer collapse or display as raw text fragments in Rich Diff
- CSS adjustments: tables inside diff blocks inherit diff background color; `pre` inside diff blocks no longer has double border-left

- All native Electron dialogs (`dialog.showMessageBox`, `alert()`) replaced with custom HTML modal dialogs:
  - **Close dialog**: "Exit without Saving", "Save for Resume & Exit", "Save As...", "Save & Exit" (context-dependent buttons for new vs. existing file)

  - **About dialog**: replaced `alert()` with HTML modal

- Zoom percentage display in status bar using `webFrame.getZoomLevel()` from preload
- isDirty false positive fix: added `suppressDirty` flag to ignore `onEditorChange` during programmatic `setContent` calls (`setContentClean` helper)

- IPC handlers refactored to register once globally with `event.sender` for multi-window support (fixes duplicate handler registration crash)
- Smoke test updated to 16 steps, added confirm-save and zoom tests

### Recovery & Backup
- Fixed recovery restore failing when backup content is empty string (`!selected.content` is true for `""`, changed to `selected.content == null`)
- When content is empty but filePath exists, reads the file from disk to populate content before restoring
- Recovery modal: Word/Excel-style list with radio selection, double-click to restore, keyboard navigation (Up/Down/Enter/Escape), delete button
- Restore from Backup accessible via File menu
- Confirmation dialog when restoring over unsaved changes
- Autosave backup content includes `originalContent` for accurate diff after restore
- Fixed search/global-search initialization after recovery by calling `triggerGlobalSearchUpdate()` post-restore

## v1.0.x (2026-02-20)

### Navigation Safety
- All in-app navigation blocked (`will-navigate`, `new-window`) to prevent losing the editor page
- Mouse back/forward buttons intercepted
- External links (http/https) open in the system browser via `shell.openExternal()` (3-layer defense: renderer click handler, `will-navigate`, `window.open` handler)
- `#anchor` links within preview/diff panes scroll to the target element within the pane (uses `CSS.escape` for robust id matching)
- Relative image/asset paths in preview resolved to `file://` URLs based on the edited file's directory (`resolveRelativePaths` for img, video, audio, source, embed, video poster)

### Multi-Instance & DnD
- Single-instance lock with `requestSingleInstanceLock()`: 2nd+ instances send args to the primary process
- `second-instance` handler creates a new `BrowserWindow` in the existing process (instead of spawning separate processes)
- New Window menu item (Ctrl+Shift+N) creates `BrowserWindow` in same process
- Drag-and-drop rewrite:
  - Default drop: opens file in a new window
  - Shift+Drop: opens file in the current window (with unsaved-changes confirmation via `showConfirmSaveDialog`)
  - DnD Insert Mode toggle (default OFF): when ON, drop inserts file content at cursor position
  - Disabled CodeMirror's built-in drop handler to prevent unwanted text insertion
- Electron 33 compatibility: uses `webUtils.getPathForFile()` instead of deprecated `file.path`
- CLI support: `mdpad.exe --help`, `mdpad.exe --version`, `mdpad.exe -- <file>` (command-line file argument for OS-level file association)
- Fixed DnD event propagation: removed `stopPropagation()` from CodeMirror drop handler so events bubble correctly
- Fixed EXE self-open bug when dropping files
- Close listener cleanup on window close to prevent memory leaks

### Editor Enhancements
- Go to Line dialog (Ctrl+G or click status bar cursor position)
- Overwrite mode: Insert key toggles between insert and overwrite mode, with block caret and `<INS>` status indicator in red
- Preview pane sticky header (matching diff pane behavior)
- Cursor line highlight transparency fix for selection visibility
- Cursor z-index fix to prevent cursor hiding behind backgrounds
- Search match count display in Find panel ("3 hits", replaced count feedback)
- Global search bar across all panes (Editor/Preview/Diff) with highlight marks and real-time hit count
- DnD insert mode toggle in the global search bar area
- Global search: replaced 500ms polling with event-based triggers (fires on editor change, replace, diff file update; debounce 200ms)

### Smoke Test
- Automated EXE smoke test via CDP (Chrome DevTools Protocol), using Node.js 22+ built-in WebSocket:
  - Recovery modal detection and restore
  - Find panel (Ctrl+F), Find Next/Previous, Replace
  - Global search
  - Select All / Delete / Undo
  - isDirty state verification
  - HTML close dialog detection and cancel
  - Confirm-save dialog infrastructure check
  - Zoom percentage verification
  - 16 test steps, all automated
- `build:full` npm script for build + smoke test + ZIP pipeline
